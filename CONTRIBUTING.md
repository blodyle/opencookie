# Contributing

OpenCookie is intentionally small. Contributions are welcome when they keep the project easier to understand, safer to use, or better documented.

Good contributions include:

- Bug fixes.
- Documentation improvements.
- Example improvements.
- Accessibility improvements.
- Next.js compatibility fixes.
- Better TypeScript support.
- Better tests.
- Smaller bundle size.
- Better developer experience.

Avoid contributions that add:

- A backend.
- Telemetry.
- A dashboard.
- Hosted services.
- Vendor scanning.
- Cookie scanning.
- Heavy dependencies.
- Legal compliance claims.
- Mandatory UI styling.

Pull requests should be small, focused, and easy to review. Add or update tests when behavior changes.
